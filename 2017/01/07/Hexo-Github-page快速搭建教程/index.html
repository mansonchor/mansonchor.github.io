Hexo + Github-page快速搭建教程

2017-01-07 21:07

记录一下自己在使用Hexo和GitHub-page搭建博客时的一些经验

如果你是一个程序员，基本上你的电脑肯定已经有了 node 和 git，这个我就不介绍了，直接上Hexo

Hexo使用

1.使用npm安装,很方便

2.初始化，cd 到你想创建Hexo项目的地方，然后

然后你会看到你当前目录里已经出现了 Hexo 的项目目录结构

3.启动本地服务，进行文章预览调试、体验

4.创建一篇新文章

至此，你本地 server 启动都能看到 source/_posts 里的 md 文章了

5.发布到 Github-page

博客内容只在本地也没用阿，我们需要有一个托管的地方，因为 Hexo 生成的都是静态文件，使用 Github-page 是比较合适的，不过前提是你要有个 github 账号

需要使用 deploy 来发布你的内容，具体的配置我不想做搬运工了，直接给官网的文档 Docs: https://hexo.io/docs/deployment.html

重点说下我在发布到 github 时遇到的一些问题:

commit的账号和我github 的账号对不上

因为我 git --global 里本身有公司的 git 账号设置，而 Hexo 发布到 github 的过程是没有传统 git 的 commit 过程的

导致当执行 hexo deploy 的时候，会直接拿 git --global 里的账号进行 commit 和 push 到 github

解决方案：单独设置 Hexo deploy git 里的账号

这样就配置了发布到 github 时会使用你 github 的账号，但是不会影响到 git --global 的配置
